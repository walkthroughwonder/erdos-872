# Audit report for Erdős 872 PR package

## 1. Verdict

**BLOCK - REQUIRES REWORK**

The package is close in tone and style, but it should not be submitted until four issues are fixed: the weak lower-bound quantifier, the undefined `gameLength`, the overstated K5 activation claim in the PR description, and the misleading prime-count variant description.

## 2. Critical issues (must fix before clicking PR button)

1. **The weak statement is almost certainly misformalized.** The Lean file states `∀ ε > 0, ∀ᶠ n, L n ≥ ε * n`, which would demand lower bounds larger than `n` for choices such as `ε = 2`, while the same file states `L n ≤ n - 1` for `n ≥ 2` ([872_v2.lean](872_v2.lean)). The public problem page phrases the first target as “At least ε n moves? (For ε>0 and n sufficiently large.)”, which should be formalized as the existence of a positive fixed constant, not as every positive constant ([Erdős Problem 872](https://www.erdosproblems.com/872)).

2. **The central game value is undefined.** `gameLength` is `noncomputable def gameLength {n : ℕ} : GamePos n → ℕ := sorry`, so the theorem statements do not yet formalize the minimax alternating game value ([872_v2.lean](872_v2.lean)). This is materially different from theorem proofs being `sorry`, because the main object in every statement, `L`, is arbitrary rather than defined ([872_v2.lean](872_v2.lean)).

3. **The PR description overstates the activation exclusion result.** The PR says the linked analysis gives “a structural argument (vertex-3-star deletion) that activation never reaches such fibers from the initial state”, but the certification only proves that all ten K5 pair activations are unsafe as a first activation move in the full activation graph and explicitly does not rule out multi-step activation strategies that first secure small-prime pairs ([pr_description.md](pr_description.md), [k5_safe_edge_certification.md](k5_safe_edge_certification.md), [k5_activation_exclusion_argument.md](k5_activation_exclusion_argument.md)). The forum draft correctly discloses this caveat, but the PR does not ([forum_post_v3.md](forum_post_v3.md)).

4. **The prime-count variant is described as a lower bound rather than an open forum question.** The forum content visible from the public thread says the primes show a maximal primitive subset can have size `π(n)` and asks whether this is smallest for all `n ≥ 2`, which would imply every game lasts at least `π(n)` moves ([Erdős forum thread](https://www.erdosproblems.com/forum/thread/872)). The Lean comment says the lower bound holds “because the set of primes ...”, but that only supports an example of a maximal primitive subset, not the universal game lower bound ([872_v2.lean](872_v2.lean)).

5. **The PR checklist claim “no hidden axioms introduced” is not honest as written.** The file contains `sorry` inside definitions, including `applyMove` proof fields and the body of `gameLength`, not just theorem proofs ([872_v2.lean](872_v2.lean)). Replace the checklist item with wording that admits definition placeholders or remove it ([pr_description.md](pr_description.md)).

## 3. Recommended edits (should fix, but not blocking)

1. **Submit as a draft PR or RFC, not a ready PR.** The contribution guide asks contributors to ensure the project builds with `lake build`, while the compile note records only a single-file `lake env lean` check and says a full build is still worth doing for project-specific linters ([CONTRIBUTING.md](ref/CONTRIBUTING.md), [872_compile_notes.md](872_compile_notes.md)).

2. **Add or discuss AMS 91.** The Lean file uses `AMS 5 11`, matching nearby combinatorics and number theory examples ([865.lean](ref/865.lean), [868.lean](ref/868.lean)). The Buddhdev manuscript classifies the problem with primary `91A46` and secondary `05D05, 11N25`, so `AMS 5 11 91` may be more accurate if the repo accepts multiple tags including 91 ([buddhdev_872.txt](buddhdev_872.txt)).

3. **Soften the Buddhdev citation because the manuscript is private.** The local manuscript supports the conditional lower bound and the restricted safe-edge hypothesis, but maintainers cannot verify a private manuscript unless it is attached, posted, or quoted enough in the PR ([buddhdev_872.pdf](buddhdev_872.pdf), [buddhdev_setup_audit.md](buddhdev_setup_audit.md)). Use “private manuscript shared with me” and avoid implying public verifiability ([pr_description.md](pr_description.md)).

4. **Replace “first game-theoretic Erdős problem” with a softer scan claim.** The exact “431 existing files” and “zero hits” claim is too brittle for a PR body because it can go stale between branch creation and review ([pr_description.md](pr_description.md)). Prefer “I did not find an existing game-theoretic Erdős file in a keyword scan” ([README.md](ref/README.md)).

5. **Confirm the submitted path is `FormalConjectures/ErdosProblems/872.lean`.** The audited file is named `872_v2.lean`, while the repo convention is one problem per file in the matching source directory ([872_v2.lean](872_v2.lean), [CONTRIBUTING.md](ref/CONTRIBUTING.md)).

6. **Avoid tagging handles unless verified in the live forum UI.** The fetched public thread content did not show `@Om_Buddhdev_sensho`, `@Jonas Silva`, `@pommeret`, or `@old-bielefelder`, although other local notes mention `Om_Buddhdev_sensho` and `Jonas Silva` from a wiki context ([Erdős forum thread](https://www.erdosproblems.com/forum/thread/872), [ai_erdos_sweep_brief.md](ai_erdos_sweep_brief.md)).

A short comment can say that this is a length game with both players adding legal elements, not the usual Maker claims and Breaker blocks game ([872_v2.lean](872_v2.lean)).

3. **Keep the forum caveat from line 34 prominent.** The forum draft’s sentence that multi-step activation remains open is the right level of caution and should also appear in the PR ([forum_post_v3.md](forum_post_v3.md)).

4. **Remove promotional phrasing aimed at AI systems.** “Accessible to Aristotle, AlphaProof, and the other formal-prover systems currently sweeping the Erdős corpus” may read as promotional to maintainers ([pr_description.md](pr_description.md)). The repo README already frames the benchmark goal neutrally ([README.md](ref/README.md)).

## 6. Buddhdev citation verdict

**needs-softening**

The local manuscript exists and matches the core conditional claim: Theorem 4.7 states that, under the restricted safe-edge hypothesis, for every fixed `0 < δ < 1/4` there is `cδ > 0` with `L(n) ≥ cδ n(log log n)^2 / log n` for sufficiently large `n` ([buddhdev_872.txt](buddhdev_872.txt)). Definition 4.5 states the restricted safe-edge hypothesis for states actually reached by the T2 activation strategy and residual slot construction ([buddhdev_872.txt](buddhdev_872.txt)). Proposition A.9 repeats the conditional T2 lower bound for activation graph states and residual slot-hypergraph states generated by the construction ([buddhdev_872.txt](buddhdev_872.txt)).

The citation is not submission-safe as a normal public reference because the manuscript is not public, and the PR currently asks reviewers to trust a private manuscript plus a public audit repository ([pr_description.md](pr_description.md)). Keep the conditional theorem only if the PR clearly says “private manuscript shared with me” and quotes enough of Theorem 4.7, Definition 4.5, and Proposition A.9 for reviewers to evaluate the statement ([buddhdev_setup_audit.md](buddhdev_setup_audit.md)).
