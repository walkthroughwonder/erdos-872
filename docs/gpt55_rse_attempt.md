# GPT-5.5 RSE attack on Erdős Problem 872, Target LB-1

TL;DR. I did not obtain a proof of the restricted safe-edge hypothesis. I found a type-level obstruction to the proposed invariant: the T2 securing move makes the small primes `a`, `c`, and `ac` unavailable in the original divisibility game, but it does not put the residual slot vertices `b`, `ab`, and `cb` into the captured-vertex set `C` of the rank-three slot hypergraph. Consequently the statement that the small primes `{13,17,19,23}` are already in `C` does not prevent the `K_4` fiber over a large prime `q` from appearing with slot-capture set empty. As concrete progress, I give a Lean-formalizable local certificate, called the dominant-capture certificate, which is a checkable sufficient replacement for the one-step safe-edge conclusion. If this certificate is verified on the T2-generated residual states, Buddhdev's Proposition A.3 through Proposition A.9 imply the same conditional lower bound, namely `L(n) >= c_delta n (log log n)^2 / log n`. The residual open claim is exactly the verification of this certificate, which is narrower and more numerical than unrestricted RSE.

## 1. Setup, state variables, invariant, witness, and target claim

Fix `0 < delta < 1/4`, put `Y := n^delta`, and let `P_Y` be the set of odd primes at most `Y`. For odd primes `a < c` in `P_Y`, put

```text
I_{a,c} := ( n/(2ac), n/(ac) ],
B_{a,c} := I_{a,c} cap P,
T := { acb : a < c, a,c in P_Y, b in B_{a,c} }.
```

For large `n`, every `b in B_{a,c}` satisfies `b > Y`, since `b > n/(2Y^2) = (1/2)n^{1-2delta} > n^delta = Y`. Thus `a`, `c`, and `b` are distinct primes, and each target `acb` lies in `(n/2,n]`. This is the target family used in Appendix A of Buddhdev's manuscript [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

The residual slot hypergraph has vertex set consisting of the slots that occur in targets,

```text
b, ab, cb,
```

and has one rank-three edge

```text
e_{a,c,b} := { b, ab, cb }
```

of weight `1` for each target `acb`. This is the scored rank-three slot hypergraph `G(n,delta)` in Appendix A [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

A finite scored slot state is a tuple

```text
sigma = (C, D_V, D_E, K, S),
```

where `C` is the captured slot-vertex set, `D_V` is the deleted slot-vertex set, `D_E` is the set of live edges deleted without scoring, `K` is the scored-edge set, and `S = sum_{e in K} w(e)` plus the weight scored by Maker captures. An edge `e` is live at `sigma` if and only if

```text
e notin D_E union K, and e cap D_V = empty.
```

The scaled potential is

```text
Q(sigma) := 8 S(sigma) + sum_{e live at sigma} Phi_sigma(e),
Phi_sigma(e) := 2^{|e cap C|} w(e).
```

For the present arithmetic residual hypergraph, `w(e)=1`, so `Phi_sigma(e)` is one of `1`, `2`, `4`, or `8`.

The restricted safe-edge hypothesis asks for the following one-round statement on the T2-generated state subspace. At every pre-Maker state `sigma` generated by the T2 activation strategy and the residual slot construction, if a positive-weight live edge exists, there is a live edge `f` such that after Maker plays `f`, every legal Breaker reply leaves `Q` at least as large as it was before Maker's move. In formula form,

```text
exists f live at sigma, forall legal Breaker replies b,
Q(sigma + f + b) >= Q(sigma).
```

Buddhdev's Definition 4.5 states this as the restricted safe-edge hypothesis, and Theorem 4.7 says that this hypothesis gives `L(n) >= c_delta n (log log n)^2 / log n` for every fixed `0 < delta < 1/4` [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

I will use the following structural invariant for the actual residual construction.

```text
Inv_res(sigma) : Prop :=
  (1) sigma is obtained from a secured-pair residual family T_* by legal slot-game moves;
  (2) every live edge e at sigma has the form e_{a,c,b} = {b,ab,cb};
  (3) for every live e_{a,c,b}, the original target acb is still legal in the divisibility game modulo the already played securing target t_0(a,c) = acb_0;
  (4) if a Breaker move in the original game is comparable with acb through b, ab, cb, or acb itself, then the corresponding slot-game move deletes the slot vertex b, ab, cb, or scores/deletes the edge e_{a,c,b};
  (5) no assertion is made that the small primes a or c are slot vertices.
```

Clause (5) is essential. The slot vertices are `b`, `ab`, and `cb`. The small primes `a` and `c` are not vertices of the residual slot hypergraph. This is already implicit in Proposition A.5, where the securing target makes `a`, `c`, and `ac` unavailable as divisibility-game move options, while the residual slot labels attached to `t=acb` are exactly `b`, `ab`, and `cb` [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

The witness I can prove safe is not an unconditional function on all `Inv_res` states. Instead, I prove a checkable local sufficient condition. For a live edge `f`, let `sigma[f]` denote the state after Maker captures `f` ordinarily, not by alternate scoring. Define the exact Maker gain

```text
G_sigma(f) := Q(sigma[f]) - Q(sigma).
```

Equivalently, if `L(sigma)` is the live-edge set before Maker moves, then

```text
G_sigma(f)
 = 8 w(f) - Phi_sigma(f)
   + sum_{e in L(sigma), e not equal f, e live after f}
       ( Phi_{sigma[f]}(e) - Phi_sigma(e) ).
```

For the arithmetic residual game, all weights are `1`, and this becomes

```text
G_sigma(f)
 = 8 - 2^{|f cap C|}
   + sum_{e in L(sigma), e not equal f, e cap D_V = empty, e notin D_E union K}
       ( 2^{|e cap (C union f)|} - 2^{|e cap C|} ).
```

After Maker captures `f`, define the largest possible unscored-edge deletion loss

```text
E_sigma(f) := max { Phi_{sigma[f]}(e) : e live at sigma[f] },
```

with `E_sigma(f)=0` if no live edge remains, and define the largest possible vertex-deletion loss

```text
V_sigma(f) := max_{v notin C(sigma[f]) union D_V(sigma[f])}
              sum_{e live at sigma[f], v in e} Phi_{sigma[f]}(e),
```

with `V_sigma(f)=0` if no uncaptured undeleted vertex lies in a live edge.

The dominant-capture certificate is the following predicate.

```text
Dom(sigma,f) : Prop :=
  f is live at sigma and G_sigma(f) >= max(E_sigma(f), V_sigma(f)).
```

The corresponding witness function is explicit whenever the certificate is nonempty:

```text
f_dom(sigma) := the lexicographically least live edge f minimizing (a,c,b)
                among all f with Dom(sigma,f).
```

This is Lean-formalizable after fixing any total order on live edges. The mathematical content is the inequality `G_sigma(f) >= max(E_sigma(f), V_sigma(f))`.

The weaker target proved below is the following.

```text
Certified-RSE(T2):
For every T2-generated pre-Maker residual slot state sigma with a positive-weight live edge,
there exists a live edge f with Dom(sigma,f).
```

This is not a proof of RSE. It is a concrete sufficient certificate that implies RSE on the certified states. It is narrower than unrestricted RSE in two senses. First, it only talks about the T2-generated states. Second, it replaces the informal safe-edge assertion by explicit finite inequalities `G >= E` and `G >= V` that can be checked from the live incidence table.

## 2. Preservation of the residual invariant

I now prove that `Inv_res` is preserved by the residual slot-game moves. This is the part of the architecture that does close unconditionally.

Assume `Inv_res(sigma)` and let `e=e_{a,c,b}` be live at `sigma`. By definition, `e={b,ab,cb}` for a residual target `t=acb` lying over a secured pair `(a,c)`. The securing target `t_0(a,c)=acb_0`, with `b_0 not equal b`, has already been played. It makes the divisibility-game moves `a`, `c`, and `ac` unavailable, and it is incomparable with `t`, since both `t_0` and `t` lie in `(n/2,n]` and are distinct. This is exactly the setup of Proposition A.5 [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

Consider Maker's ordinary capture of a live edge `e`. In the slot game, this adds `b`, `ab`, and `cb` to `C`, adds `w(e)=1` to `S`, and removes `e` from the live set. It does not create any new live edge. Every remaining live edge was already a live edge before the move, so it still has the form `{b',a'b',c'b'}`. The correspondence with a legal residual target is preserved because capturing a slot vertex represents making that divisor unavailable in the original game. Hence clauses (1) through (4) of `Inv_res` remain true. Clause (5) is a type statement and is unchanged.

Consider Maker's alternate-scoring move on a live edge `e`. This adds `e` to `K` and increases `S` by `1`, without changing `C` or `D_V`. Again no new live edge is created, and every remaining live edge was already a valid residual target edge. Thus `Inv_res` is preserved.

Consider Breaker's vertex deletion. If Breaker deletes an uncaptured slot vertex `v`, the slot state adds `v` to `D_V`, and every live edge containing `v` ceases to be live. No new live edge is created. For any remaining live edge `e_{a,c,b}`, none of its slots was deleted, and the original target `acb` is still legal with respect to the modeled harmful divisors. The slot incidence claims in Proposition A.5 give the exact correspondence: deleting `b` kills precisely the residual targets with the same large prime `b`, deleting `ab` kills precisely the residual targets whose attached slot set contains `ab`, and deleting `cb` is identical with `ab` after replacing `a` by `c` [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf). Thus `Inv_res` is preserved.

Consider Breaker's unscored edge deletion. This adds one live edge to `D_E`, without changing any vertex set. Every remaining live edge keeps its previous residual target interpretation. Hence `Inv_res` is preserved.

Consider Breaker's scored-edge reply. This adds one live edge to `K` and increases `S` by `1`. It does not change `C` or `D_V`, and it creates no new live edge. Hence `Inv_res` is preserved.

Therefore `Inv_res` is closed under every residual slot-game extension. The same proof applies to graph activation states after replacing a rank-three edge `{b,ab,cb}` by a pair edge `(a,c)` and replacing the slot labels by the graph endpoints and pair label used in Proposition A.6 [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

## 3. The proposed `K_4`-avoidance invariant is not valid as stated

The suggested invariant says, informally, that before four small primes `{13,17,19,23}` can form a `K_4` of weighted shadow edges sharing a common right slot `q`, all four small primes are already in `C`, because they were committed as left vertices in Phase 1.

This does not type-check in the residual slot hypergraph. In `G(n,delta)`, the slot vertices of the edge associated to `acq` are

```text
q, aq, cq.
```

The small primes `a` and `c` are not vertices of this residual hypergraph. Thus the assertion `13 in C` is not an assertion about the residual slot state unless `C` is redefined to include non-slot divisibility-game objects. Buddhdev's Appendix A does not define it that way. It defines the residual slot labels for `t=acb` as `b`, `ab`, and `cb`, while the securing target only makes `a`, `c`, and `ac` unavailable as move options [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

Here is the concrete consequence. Suppose the six pairs among `{13,17,19,23}` have been secured by earlier targets `acb_0`, and suppose `q in (n/442,n/437]` is a prime distinct from the six securing primes. For each pair `a<c` among `{13,17,19,23}`, the target `acq` remains a valid residual target unless one of its residual slots `q`, `aq`, or `cq`, or the exact target `acq`, has been played. The previous securing target `acb_0` does not equal `q`, `aq`, `cq`, or `acq`, and it is incomparable with `acq`, since both are distinct elements of `(n/2,n]`. Therefore the residual edge

```text
e_{a,c,q} = {q, aq, cq}
```

can be live with `|e_{a,c,q} cap C| = 0`, even though the original small primes `a` and `c` have already been committed in the activation sense.

This refutes the proposed invariant as a proof device. It does not by itself construct a counterexample in the exact T2-generated state space, because one still has to prove that the activation strategy can be forced to secure those six pairs and that the residual strategy can be forced through the exact isolating sequence. It does show that the advertised reason for excluding the `K_4` fiber is insufficient. A correct invariant must speak about the slot vertices `q`, `13q`, `17q`, `19q`, and `23q`, not about the small primes `13`, `17`, `19`, and `23`.

The `K_4` fiber obstruction from Proposition A.2 remains the relevant local warning. For primes `13,17,19,23`, the pair products lie between `221` and `437`, and a prime `q in (n/442,n/437]` lies in all six intervals `B_{a,c}` for sufficiently large `n`. The six residual edges form a `K_4` fiber over the common slot `q` [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf). After two captures and two unscored edge deletions, one reaches the local state with live edges

```text
e_{17,19} = {q,17q,19q},
e_{17,23} = {q,17q,23q},
```

and with all five slot vertices `q`, `13q`, `17q`, `19q`, and `23q` captured. At that state each live edge has potential `8`, Maker's own score gain on either edge is `8-8=0`, and Breaker can delete the other live edge without scoring, lowering `Q` by `8` [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

The important point is not merely that this state exists in the full residual arithmetic game. The important point is that activation of the small primes does not change the slot-capture count of the six `q`-fiber edges. Thus an RSE proof needs an invariant that controls the evolution of the slot vertices inside each large-prime fiber.

## 4. Exact one-round inequalities for the dominant-capture certificate

I now prove the promised local safe-edge lemma. This is the part intended to be directly formalizable.

Let `sigma` be a finite scored rank-three slot state satisfying `Inv_res`, and let `f` be a live edge. Maker plays `f` by ordinary capture. Let `sigma_M := sigma[f]` be the state immediately after Maker's move and before Breaker replies. Define

```text
G := G_sigma(f) = Q(sigma_M) - Q(sigma).
```

Assume `Dom(sigma,f)`, namely

```text
G >= E_sigma(f) and G >= V_sigma(f).
```

I check Breaker's three legal replies.

### 4.1 Breaker deletes a vertex

Let Breaker delete an uncaptured undeleted vertex `v` at `sigma_M`. If `v` lies in no live edge at `sigma_M`, then no live edge is removed, so

```text
Q(sigma_M + delete_vertex(v)) = Q(sigma_M) = Q(sigma) + G >= Q(sigma).
```

If `v` lies in at least one live edge, then the only change in `Q` caused by deleting `v` is the removal of the potential contribution of the live edges containing `v`. The score `S` does not change. Hence

```text
Q(sigma_M + delete_vertex(v))
 = Q(sigma_M) - sum_{e live at sigma_M, v in e} Phi_{sigma_M}(e).
```

By definition of `V_sigma(f)`,

```text
sum_{e live at sigma_M, v in e} Phi_{sigma_M}(e) <= V_sigma(f) <= G.
```

Therefore

```text
Q(sigma_M + delete_vertex(v)) >= Q(sigma_M) - G = Q(sigma).
```

This proves the vertex-deletion case.

### 4.2 Breaker deletes one edge without scoring

Let Breaker delete a live edge `e` without scoring at `sigma_M`. If no live edge exists, this reply is illegal. Otherwise the score `S` does not change, and the only change in `Q` is the removal of `Phi_{sigma_M}(e)`. Therefore

```text
Q(sigma_M + delete_edge(e))
 = Q(sigma_M) - Phi_{sigma_M}(e).
```

By definition of `E_sigma(f)`,

```text
Phi_{sigma_M}(e) <= E_sigma(f) <= G.
```

Hence

```text
Q(sigma_M + delete_edge(e)) >= Q(sigma_M) - G = Q(sigma).
```

This proves the unscored edge-deletion case.

### 4.3 Breaker makes a scored-edge reply

Let Breaker score a live edge `e` at `sigma_M`. The edge `e` is moved to `K`, and `S` increases by `w(e)`. The change in `Q` caused by this Breaker reply is

```text
8 w(e) - Phi_{sigma_M}(e)
 = (8 - 2^{|e cap C(sigma_M)|}) w(e).
```

Since the rank is at most `3`, one has `|e cap C(sigma_M)| <= 3`, and therefore

```text
8 - 2^{|e cap C(sigma_M)|} >= 0.
```

Thus

```text
Q(sigma_M + score_edge(e)) >= Q(sigma_M) = Q(sigma) + G >= Q(sigma).
```

This proves the scored-edge reply case.

Combining the three cases gives the following lemma.

```text
Lemma (dominant-capture implies safe edge).
If sigma is a finite rank-at-most-three scored slot state and f is a live edge satisfying
G_sigma(f) >= max(E_sigma(f), V_sigma(f)), then ordinary capture of f is a legal Maker move
such that every legal Breaker reply leaves Q at least as large as it was before Maker's move.
```

No symmetry is used in this proof. The witness `f_dom(sigma)` is the least edge satisfying the displayed inequality. All quantities are finite sums over the live incidence table.

## 5. Why the naive max-gain witness does not close RSE

The recommended witness in the prompt was to take a live edge maximizing the alternate-scoring gain

```text
8 w(f) - Phi_sigma(f).
```

In the unweighted rank-three residual game this quantity equals `8 - 2^k`, where `k=|f cap C|`. It is at most `7`. An unscored deletion of a different live edge can remove potential as large as `8`. Therefore alternate scoring alone cannot prove the unscored edge-deletion case. The inequality needed for an unscored deletion is

```text
8 - Phi_sigma(f) >= max_{e live after f} Phi(e),
```

if Maker uses alternate scoring and gains no additional incident-edge doubling. This inequality is impossible whenever another live edge of potential `8` remains and `Phi_sigma(f) >= 1`.

Ordinary capture can gain more than alternate scoring because it increases the potential of other live edges sharing newly captured slots. The exact additional gain is

```text
sum_{e not equal f} (2^{|e cap (C union f)|} - 2^{|e cap C|}).
```

The dominant-capture certificate records precisely how large this additional gain must be. It must dominate not only the largest single live-edge potential, but also the largest weighted star around any vertex that Breaker may delete after Maker's move.

This explains the role of the `K_4` fiber. In the final two-edge `K_4` state, both remaining live edges already have all three vertices in `C`. If Maker captures one of them, then

```text
G = 8 - 8 + 0 = 0,
E = 8,
V = 0,
```

so `Dom(sigma,f)` fails for both live edges. If Maker alternate-scores one of them, the gain is also `0`, and Breaker deletes the other live edge for a loss of `8`. Thus any valid T2 invariant must prevent this precise slot-captured two-edge configuration, not merely record that the underlying small primes were activated earlier.

## 6. The certified replacement and the implication for Buddhdev's T2 theorem

Define the certified restricted safe-edge hypothesis as follows.

```text
Certified-RSE(T2):
For every pre-Maker state sigma generated by the T2 activation strategy and the residual slot
construction, if sigma has a positive-weight live edge, then there exists a live edge f with
Dom(sigma,f).
```

This hypothesis is concrete. Given a finite state, one computes `G_sigma(f)`, `E_sigma(f)`, and `V_sigma(f)` for each live edge `f`, and checks whether at least one live edge satisfies

```text
G_sigma(f) >= E_sigma(f),
G_sigma(f) >= V_sigma(f).
```

The lemma in Section 4 proves that `Certified-RSE(T2)` implies Buddhdev's restricted safe-edge hypothesis for the corresponding states. Therefore Proposition A.3 applies to the activation graph states and to the residual rank-three slot states. Proposition A.3 says that, under the restricted safe-edge hypothesis, the final claimed or scored weight is at least one eighth of the initial total edge weight [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

The rest of Buddhdev's Appendix A is then unchanged. Proposition A.4 gives initial target mass

```text
W_0 := sum_{a<c, a,c in P_Y} |B_{a,c}| >>_delta n (log log n)^2 / log n,
```

using the prime number theorem uniformly in the fixed-ratio intervals `(n/(2ac), n/(ac)]` and Mertens' theorem for primes [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf). In the notation requested by the prompt, the implicit constant may be taken from a fixed `kappa_delta > 0` such that

```text
pi(X) - pi(X/2) >= kappa_delta X / log n
```

for all `X = n/(ac) >= n^{1-2delta}` and all sufficiently large `n`. Then

```text
W_0 >= (kappa_delta/3) n (log log n)^2 / log n
```

for all sufficiently large `n`, after absorbing the bounded sum `sum_{p in P_Y} p^{-2}` and the `O_delta(1)` term in Mertens' theorem. Buddhdev cites the needed prime number theorem and Mertens inputs as Hardy-Wright Theorems 6 and 7 in the surrounding argument [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf).

Proposition A.7 bounds the activation off-model deletion budget by

```text
E <<_delta Y^4 / log^4 Y
  = O_delta(n^{4delta} / log^4 n)
  = o_delta(n (log log n)^2 / log n),
```

because `4delta - 1 < 0` [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf). Proposition A.8 then gives residual mass on secured pairs

```text
M >>_delta n (log log n)^2 / log n.
```

Finally, Proposition A.9 compares the residual divisibility game on the secured pairs with the scored rank-three slot hypergraph, and the one-eighth finite capture bound gives at least `M/8` residual scored moves [Buddhdev manuscript](https://www.sensho.xyz/papers/erdos-872.pdf). Thus `Certified-RSE(T2)` implies the same conclusion as Theorem 4.7:

```text
L(n) >= c_delta n (log log n)^2 / log n
```

for all sufficiently large `n`, where one may take `c_delta` to be any positive constant below the product of the constants supplied by Proposition A.4, Proposition A.8, and the final factor `1/8` in Proposition A.3.

## 7. Residual conditional claims for future work

The proof above leaves the following residual claims.

1. The full restricted safe-edge hypothesis remains unproved. I did not find a valid invariant that excludes the `K_4` fiber final state from the T2-generated residual state space.

2. The proposed invariant involving the small primes `{13,17,19,23}` is invalid as stated. The residual slot-capture set `C` contains slots such as `q` and `13q`, not the small prime `13`. Activation of `13` as a left object does not imply `13q in C`.

3. A correct `K_4`-exclusion invariant must constrain the slot-level fiber over each large prime `q`. A plausible form is: no T2-generated residual pre-Maker state contains two live edges of potential `8` in the same `q` fiber after the other four edges of a `K_4` fiber have been scored or deleted in the Proposition A.2 pattern. I have not proved this.

4. The checkable replacement left open is `Certified-RSE(T2)`. It asks for a live edge `f` satisfying two explicit inequalities:

```text
G_sigma(f) >= E_sigma(f),
G_sigma(f) >= V_sigma(f).
```

This is a finite calculation on each state, but the number of states is enormous. A future proof should exploit the fiber decomposition by large primes `b`, where each fiber is a graph on small primes and each hyperedge is `{b,ab,cb}`.

5. The scored-edge Breaker reply is harmless in every rank-at-most-three state, because its change in `Q` is `8w(e)-2^{|e cap C|}w(e) >= 0`. The only genuinely difficult cases are vertex deletion and unscored edge deletion.

6. The asymptotic inputs are not the bottleneck. The mass estimate, deletion budget, residual embedding, and final implication are already supplied by Proposition A.4 through Proposition A.9. The missing issue is purely finite and combinatorial inside the residual slot game.

## 8. Conclusion

I cannot honestly claim an unconditional proof of RSE. The core reason is that the advertised T2 invariant confuses activation of small primes in the divisibility game with capture of residual slot vertices in the hypergraph. Once the types are separated, activation does not by itself prevent a `K_4` fiber over a large prime `q` from having all six residual edges initially at slot potential `1`.

The concrete output is a replacement certificate. For each generated state `sigma`, compute the exact ordinary-capture gain `G_sigma(f)` and the two possible Breaker losses `E_sigma(f)` and `V_sigma(f)`. If some live edge satisfies `G_sigma(f) >= max(E_sigma(f),V_sigma(f))`, then that edge is safe. If this certified condition can be proved for all T2-generated pre-Maker states, Buddhdev's Proposition A.3 through Proposition A.9 make Theorem 4.7 unconditional and give

```text
L(n) >= c_delta n (log log n)^2 / log n.
```

The riskiest remaining step is to prove the certified condition from a genuine T2 slot-level invariant, especially in fibers that contain dense graphs on four or more small primes.
